
**最终资产权重：0.90（极高价值资产）**

## 二、攻击路径利用成功率评估

### 2.1 各攻击路径成功率量化

| 攻击路径 | 成功率 | 置信度 | 加权成功率 |
|---------|--------|--------|-----------|
| 支付金额篡改 | 0% | 95% | 0% |
| 鉴权绕过 | 8% | 70% | 5.6% |
| SQL注入 | 0% | 90% | 0% |
| SSRF攻击 | 35% | 60% | 21% |
| HTTP请求走私 | 25% | 50% | 12.5% |
| JWT令牌破解 | 15% | 40% | 6% |

### 2.2 综合攻击成功率

```
综合攻击成功率 = Σ(各路径加权成功率) / 路径数量
               = (0% + 5.6% + 0% + 21% + 12.5% + 6%) / 6
               = 45.1% / 6
               = 7.52%
```

**最终攻击成功率：7.52%（低成功率，但存在潜在突破点）**

## 三、动态风险得分 (DRS) 计算

### 3.1 基础风险因子

| 因子 | 值 | 说明 |
|-----|-----|------|
| 资产权重 (A) | 0.90 | 支付数据为核心资产 |
| 攻击成功率 (S) | 7.52% | 当前WAF防护下 |
| 暴露面系数 (E) | 0.85 | 仅暴露2个API端点 |
| 防护强度系数 (P) | 0.95 | 商业云WAF严格模式 |
| 业务影响系数 (I) | 0.95 | 支付篡改导致直接资金损失 |

### 3.2 DRS计算公式

```
DRS = 100 × A × (1 - (1-S) × E × P) × I

代入数值：
DRS = 100 × 0.90 × (1 - (1-0.0752) × 0.85 × 0.95) × 0.95
    = 100 × 0.90 × (1 - 0.9248 × 0.85 × 0.95) × 0.95
    = 100 × 0.90 × (1 - 0.7468) × 0.95
    = 100 × 0.90 × 0.2532 × 0.95
    = 100 × 0.2164
    = 21.64
```

### 3.3 风险等级判定

| DRS范围 | 风险等级 | 当前值 | 判定 |
|---------|---------|--------|------|
| 0-20 | 低风险 | - | - |
| 21-40 | **中低风险** | **21.64** | **✓** |
| 41-60 | 中风险 | - | - |
| 61-80 | 中高风险 | - | - |
| 81-100 | 高风险 | - | - |

**最终DRS：21.64（中低风险）**

## 五、精算结论与建议

### 5.1 承保建议

**建议承保**，但需附加以下条件：

| 条件 | 说明 |
|-----|------|
| 承保范围 | 仅覆盖支付金额篡改导致的直接资金损失 |
| 除外责任 | 不覆盖SSRF、HTTP请求走私等间接攻击 |
| 观察期 | 90天，期间需完成安全加固 |

### 5.2 风险缓解建议

1. **短期（1-7天）**：
   - 对login端点实施IP白名单访问控制
   - 部署WAF规则更新监控机制

2. **中期（1-4周）**：
   - 实施支付金额服务端校验（与订单数据库比对）
   - 部署内部API网关，实施微服务间鉴权

3. **长期（1-3月）**：
   - 迁移至更安全的支付架构（如Tokenization）
   - 实施持续安全监控与威胁情报集成

### 5.3 再保险建议

鉴于支付业务的敏感性，建议将保额的30%进行再保险分保，以分散风险。
